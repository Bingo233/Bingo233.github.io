Hello Python

发表于 2016-03-04   |   分类于 技术   |  

简述

首先，我学习Python的动机是 爬虫，因为之前的比赛中接触过一个有关空气污染的大数据项目，还有舍友在做课设时用爬虫爬取全济南市的公交站信息。
其次，现在大数据已经逐渐流行起来，而爬虫是获取数据的最有效手段。

学习收获

从一开始下载PyCharm，配置Python环境变量，到在W3CSCHOOL上学习语法、数据类型等，只花费了两天时间； 后来又认识到Python版本的问题，调查后知道了Python2/3的区别，特别是在爬虫方面，

原本爬虫的类库都是支持Python2.7的，在Python3出现后，仍有部分库不支持Python3。

Python爬虫最为有名的Scrapy框架也只支持Python2。

Python2中的urllib、urllib2在Python3中合并为urllib。

Python2中用cookielib.CookieJar()，Python3中用http.cookiejar.CookieJar()。

练习阶段

爬取简单网页信息

主要是利用urllib请求网页，必要时可以用CookieJar自动保存Cookie信息。

提取简单文字、图片

第一种方法： 用正则表达式提取相关信息，需要引入re这个库，主要方法：findall, search, match, sub；
第二种方法： 在Scrapy框架中利用XPATH可以方便的提取标签及其属性、内容~

爬取特殊的网站信息

比如Bilibili的网站，直接爬取所得的HTML只是包括一个网页框架，通过 Firefox-F12-网络可以看到，网站的主要信息是通过JS分部加载的，而网页的图片URL、文本等信息是通过请求一个JSON数据包获取的。

针对这类网站：
（1）直接仿照网站请求相应的JSON（较快） (2)模拟浏览器加载完毕后，获取网页内容（较慢）

注：Bilibili可以申请相应的 api-key，可以通过API访问Bilibili的数据库

爬取项目经验

舍友接到了爬虫的项目，要爬取https://www.tripadvisor.co.uk上所有伦敦区的Hotel的酒店信息以及所有评论的信息。 爬取总共1063的酒店信息并不难，所有评论预计有60W条，访问MySQL数据库用的是MySQLdb的库~

难点的解决：

Problem 1: 在评论过长的情况下，只有点击 –more–才会在cookie中加载全部评论信息！

用PhantomJS工具，模拟浏览器访问界面，并且可以插入运行JS，实现在开发者模式中调试的功能。
最后实现的效果是，加载页面后如果有弹窗就关闭，并模拟点击more加载全部数据，等待加载完成后，输出全部页面内容，返回到Python代码中进行处理~

Problem 2: 为爬取酒店设置断点

因为网站结构比较复杂，一开始程序崩的比较严重，就在酒店列表中设置了current_page这一字段对应于total_page，只有成功爬取时才会+1；特殊情况下，成功爬去但不会+1，重复爬取的界面可以后期在数据库中去重~

Problem 3: 用多线程加快爬取速度

用Python内置的threading库实现多线程，然而速度并没有提升…原因是threading虽然实现了多线程，可是并没有并行处理，只是分时占用一个CPU~
之后用到了threading的增强版———multiprocessing库，可以真正利用电脑的多CPU进行并行处理~

Problem 4: 因为是英国的网站，学校网很渣，即便是在阿里云上也很慢（况且学生版只有一个CPU），不得不在调试好程序之后在雇主那边运行； 应雇主的要求，如果能两台电脑都用上就好了 Orz~

之后将自动更新断点的酒店列表上传至服务器的数据库，将酒店分好段，在雇主的两台Mac上同时爬存在本地，最后合并数据库~

其他经验

压缩

注：在爬取文件url后进行下载时，可能会遇到文件压缩的问题~
一种方法是伪造浏览器header，在‘Accept-Encoding’字段不设置‘gzip’，服务器返回文件时一般就不会主动压缩~

第二种方法就是用gzip.decompress(data)来手动解压缩~

编码

读取的文本流最好进行转码，利用encode(‘utf8’)或者encode(‘gbk’)就能解决问题。
